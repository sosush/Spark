# Spark - Intelligent Project Ideation & Portfolio Suite

**Spark** is a high-performance technical ecosystem designed to bridge the gap between project management and architectural evolution. It transforms fragmented development history into a cohesive, **3D spatial constellation**, utilizing the **Llama 3.3 70B** model via **Groq** to provide autonomous technical mentorship, strategic roadmapping, and recursive project synthesis.

## Core Capabilities

### Spatial Visualization Engine
Utilizing WebGL-powered physics, Spark visualizes your project portfolio as a 3D neural network. 
*   **Domain Interconnectivity:** Nodes connect via a "spiderweb" logic based on shared technology stacks and heuristic classifications.
*   **Minimalist Interface:** A high-precision Three.js environment optimized for fluid navigation and node interaction.

### Strategic Ideation Lab
The "Lab" serves as a dedicated incubator for technical conceptualization.
*   **Neural Collision:** Synthesize revolutionary project concepts by intersecting existing technical domains using Llama 3.3 70B.
*   **Autonomous Roadmapping:** Instantly convert concepts into active projects with AI-generated MVP paths, architecture blueprints, and technology recommendations.

### Integrated Asset Vault
A sophisticated management hub for professional repositories and manual documentation.
*   **GitHub Synchronization:** Deep-parsing of repository metadata, including automated README extraction and language detection.
*   **Direct Documentation Control:** Integrated Markdown editor with real-time rendering for professional-grade manifests.
*   **AI Technical Critique:** On-demand architectural audits providing specific, high-level optimization strategies.

### Identity & Credential Management
A persistent, glass-morphic profile hub tracking your technical trajectory.
*   **Document Hosting:** Integrated PDF resume preview and secure asset hosting via Supabase Storage.
*   **Neural Node Linker:** Dynamic social and professional uplink management for cross-platform visibility.

## Deployment Protocols

### Environment Variables
To initialize the system, the following secrets must be configured:
*   `NEXT_PUBLIC_SUPABASE_URL`: Endpoint for database interaction.
*   `NEXT_PUBLIC_SUPABASE_ANON_KEY`: Client-side access token.
*   `NEXT_PUBLIC_BACKEND_URL`: Production endpoint for the FastAPI Brain.
*   `GROQ_API_KEY`: Token for high-speed LLM inference.
*   `SUPABASE_KEY`: Service-role key for backend database management.

### Cloud Orchestration
*   **Interface:** Deployed via **Vercel** with automatic monorepo subdirectory detection.
*   **Brain:** Containerized via **Docker** and hosted on **Hugging Face Spaces** (Port 7860).
*   **Database:** Hosted on **Supabase** with enabled Row-Level Security (RLS).
